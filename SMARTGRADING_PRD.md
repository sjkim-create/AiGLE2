## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-01`
- **화면명**: Smart Grading
- **대상 사용자**: AiGLE 개발자가 이용하는 프롬프트 테스트 및 관리 화면
- **비즈니스 목적**: 등록할 문제를 선택하고 학생 답안지를 업로드, 시스템 프롬프트를 AI 모델에 전달하여 서비스에서 원하는 채점 결과를 얻고 관리할 수 있게 한다. 

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt Studio 화면 진입 시, 입력 파이프라인(왼쪽)과 AI 설정(오른쪽) 사이드바가 노출된다.
2. (Step 1) 등록문항 선택 : 등록된 과제 문항 중 하나를 선택한다. 
   해당 문항의 '문제', '모범답안', '채점기준' 데이터 BACK에서 로드되어 처리 되도록 한다. 
3. (Step 2) 채점할 학생의 답안 이미지를 업로드한다.
4. (Step 3) 기본값으로 적용된 OCR 프롬프트와 채점 프롬프트(System Prompt)을 수정 또는 그대로 사용한다.
5. (오른쪽 사이드바) 적용할 AI 모델(GPT/Gemini)과 파라미터(Temperature, Top-p, Max Output tokens, thinking_config 등)를 설정한다.
6. GPT, Gemini 모델에 따라 파라미터 항목이 분기되어 노출되도록 한다.
7. 하단의 `AI 실행` 버튼을 눌러 채점을 진행하고, 결과를 확인한다. 파악된 결과는 [아카이브에 저장]을 눌러 보존한다.
8. 보존된 결과는 Prompt 아카이브 메뉴에서 확인 가능하다.

## 2. 화면 요소 및 기능 (Elements & Functions)

### 2.1 등록 문항 선택 영역 (Step 1)

- **주요 요소:**
  - `Select Box`: 채점할 테스트 문항 선택 (예: '[국어] 문학 지문 분석', '[수학] 2차 방정식 풀이')
  - `선택 문항 정보 패널`: 선택 시 숨겨져 있던 [문제], [모범답안], [채점기준] 데이터는 BACK에서 로드되어 처리 되도록 한다.  
- **기능 정책:**
  - 등록된 과제의 과제명이 노출되어야 함
  - 필수 입력 조건 (선택이 완료되어야 AI 실행 버튼 활성화 조건 1 충족)
  - 문항을 변경하면 표시된 기존 결과 데이터 및 로드 상태가 초기화됨

### 2.2 학생 답안 이미지 업로드 영역 (Step 2)

- **주요 요소:**
  - `Drag & Drop Zone / File Input`: 클릭 시 파일 탐색기 열림, 파일 드래그로 업로드 가능, 파일 첨부 시 이미지 미리보기로 전환
  - `미리보기 이미지`: 업로드된 학생 답안 이미지 표시 영역
- **기능 정책:**
  - 필수 입력 조건 (이미지가 등록되어야 AI 실행 버튼 활성화 조건 2 충족)
  - 이미지(JPG, PNG) 또는 PDF 파일만 업로드 가능하며, 이미지는 1개만 업로드 가능
  - 이미지가 변경되면 이전 이미지는 없어지고, 추가된 이미지로 대체한다. 
  - 삭제버튼 클릭 시 업로드된 이미지가 삭제되고, 초기 상태로 돌아간다.
  
### 2.3 시스템 프롬프트 설정 (Step 3)

- **주요 요소:**
  - `OCR 프롬프트 TextArea`: 기본 프롬프트 노출, 내용이 없으면 "프롬프트 내용이 없습니다" 텍스트 노출, 영역에 포커스 시 텍스트가 사라짐
  - `채점 프롬프트 (System Prompt) TextArea`: 기본 시스템 프롬프트 노출, 내용이 없으면 "프롬프트 내용이 없습니다" 텍스트 노출, 영역에 포커스 시 텍스트가 사라짐
  - `[현재 내용을 기본값으로 저장] 버튼`: 현재 작성된 textarea영역의 프롬프트 내용을 향후 서비스 이용 시의 기본값으로 DB(LocalStorage)에 저장
  - `[초기값으로 복구] 버튼`: 직전에 저장했던 기본값으로 텍스트 영역 내용을 복구 (저장 이력이 없을 시 시스템 최초 초기값 로드)

- **기능 정책:**
  - 사용자가 직접 타이핑하여 내용을 실시간으로 편집할 수 있음
  - **기본값 저장 시**: "현재 프롬프트 내용이 기본값으로 저장되었습니다. 이후 서비스 이용 시 해당 내용이 기본값으로 자동 반영됩니다." 토스트 메시지 노출
  - **초기값 복구 시**: "프롬프트 내용이 초기화 되었습니다." 토스트 메시지 노출

### 2.4 AI 모델 선택 및 파라미터 설정 (우측 사이드바)

- **주요 요소 (모델 선택):**
  - `GPT 모델` 그룹: GPT-4o mini, GPT-5.3 Instant, GPT-5.4 Thinking, GPT-5.4 Pro 버튼
  - `Gemini 모델` 그룹: Gemini 3.1 Pro, Gemini 3.1 Flash-Lite 버튼

- **주요 요소 (파라미터 설정):**
  - 모델에 따른 조건부 랜더링 기능 도입
  - `temperature` (Range 0~1): 창의성/일관성 조절
  - `top_p` (Range 0~1): 단어 선택 폭 조절
  - `max_tokens` (Range 100~8192): (GPT, Gemini에 따라 레이블 이름 분기)
  - `thinking` (토글): GPT 모델 한정 노출 (boolean 활성/비활성 처리)
  - `thinking_config` (버튼): Gemini 모델 한정 노출 (Minimal, Low, Medium, High 4단계 버튼형), Medium이 기본값
- **기능 정책:**
  - 선택한 모델(GPT vs Gemini)에 대응하는 API 스펙을 직관적으로 조절할 수 있게 구성

## 3. 기능 수행 및 상태별 대응 (State & Interaction)

| 상태 | 발생 조건 | UI/UX 동작 및 메시지 |
| :--- | :--- | :--- |
| **Pending (대기)** | 화면 최초 진입 | 입력되지 않은 정보(문항 또는 탭안)가 있을 경우, 버튼 비활성화 |
| **Loading (채점 중)** | AI 실행 버튼 클릭 시 | 하단 버튼이 텍스트 대신 `AI 채점 중...` 텍스트로 변경, 버튼 중복 클릭 방지 차원 Disabled 처리 |
| **Success (결과 반환)** | AI API 호출 응답 완료 시 | 애니메이션과 함께 왼쪽 파이프라인 하단에 '📊 분석 완료' 결과 카드 노출 |

### 3.1 채점 결과 카드 (분석 완료 상태)

- **표시 항목:**
  - `OCR 변환 텍스트`: 펜 데이터 혹은 이미지에서 문자로 변환된 Raw Text
  - `AI 채점 결과`: 평가 등급, 피드백을 포맷팅된 형태 (원문) 로 표시
  - `채점 시간`: 예) 2.42s
  - `사용 토큰 정보`: 입력 / 출력 / 총합 수치 출력
  - `예상 비용`: US Dollar 기반 소수점, 버튼 클릭 시 원화(KRW)로 실시간 환율 연산되어 노출 (↻ 버튼으로 갱신)

- **아카이브 저장 기능:**
  - 결과 창 우측 상단의 `아카이브에 저장` 버튼을 누르면 분석내용 저장
  - 저장 완료 시 버튼 텍스트가 `✓ 아카이브 저장됨` 으로 변경됨과 동시에 다시 클릭할 수 없게 Disabled(잠금) 처리
  - '테스트 결과가 아카이브에 저장되었습니다...', 토스트 메시지 출력

## 4. 비즈니스 로직 (Business Rules)

- **입력 유효성 판단**: 좌측의 `selectedAssignId` (문제) 와 `studentImage` 가 모두 존재할 때만 실행 버튼 활성화
- **모의 데이터 채점 연산**: 현재 버전에서는 MockData 기반의 Promise(setTimeout 2초간 딜레이) 처리하여 결괏값을 노출 (추후 실제 AI API와 연동)
- **비용 환율 환산**: `$ 0.00085`와 같이 반환된 미국 기준 비용에 `exchangeRate`(기본 1350원 기준/새로고침 시 난수 반영)을 곱하여 최종 원화로 노출

## 5. 접근성 및 UI 가이드

- **반응성(피드백)**: 아카이브 저장 버튼 클릭 및 AI 채점 중에 Loading 상태를 시각화하여 사용자의 이중 동작(더블 체크) 방지
- **섹션 분리**: 왼쪽 영역 (입력 및 결과), 오른쪽 영역 (API 설정 및 실행) 으로 레이아웃을 구분하여 정보의 위계를 최적화
