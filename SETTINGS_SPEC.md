# 환경설정(SETTINGS) 메뉴 및 기능 명세서

본 문서는 AiGLE 프로젝트의 환경설정 메뉴 구성, 기능, 모달창 노출 조건, 로직 및 예외 처리 등에 대한 상세 명세입니다.

---

## 1. 메뉴 구성 및 화면 레이아웃 (Layout)

환경설정은 사용자가 자신의 프로필을 클릭하여 진입하며, 독립적인 **카드형 레이아웃**으로 구성됩니다.

### 1.1 내비게이션 진입 및 전환

- **진입점**: 사이드바 상단 사용자 프로필 클릭 시 나타나는 드롭다운 메뉴 (내정보, 환경설정, 로그아웃)에서 [환경설정] 클릭.
- **설정 모드(isSettingsMode)**: 진입 시 전용 사이드바(내 정보, 고객센터, 환경설정)로 전환

### 1.2 메인 화면 구성 (Cards)

| 기능 영역               | 개요 및 노출 정보                         | 주요 액션                                |
| :---------------------- | :---------------------------------------- | :--------------------------------------- |
| **NeoStudio2Lite 관리** | 소프트웨어 설치 여부 및 버전 정보         | [최신 버전 다운로드]                     |
| **펜 펌웨어 업데이트**  | 연결된 펜 개수 및 업데이트 필요 상태 요약 | [펜 목록 및 업데이트 확인] (모달 오픈)   |
| **기기 초기화**         | 위험성 안내 및 전체 초기화 기능           | [연결된 펜 전체 초기화] (확인 모달 오픈) |

## 2. 주요 기능 및 모달창 (Modals)
### 2.1 NeoStudio2Lite 관리
- **노출 조건**: 카드에 상세 메시지 표시로 최신 버전이 아닌 경우에 버튼 활성화 
- **화면 구성**:
  - 설치되지 않음: 현재 버전 "설치되지 않음" / 최신 버전 "X.X.X" -> [최신 버전 다운로드] 버튼 활성화
  - 업데이트 필요: 현재 버전 "X.X.X" / 최신 버전 "X.X.X" -> [최신 버전 다운로드] 버튼 활성화
  - 설치됨: 현재 버전 "X.X.X" / 최신 버전 "X.X.X" -> [최신 버전 다운로드] 버튼 비활성화

- **주요 액션**:
  - [최신 버전 다운로드]: NeoStudio2Lite 최신 버전 다운로드.

### 2.2 펜 펌웨어 업데이트 모달 (isFirmwareModalOpen)
- ** 카드 화면 구성 : 
  - 연결된 펜 : ** 개
  - 업데이트 필요: 업데이트가 필요한 펜 있는 경우 상태 표시 
  
- **노출 조건**: 사용자가 [펜 목록 및 펌웨어 업데이트] 버튼을 클릭했을 때.

- **모달 화면 구성**:
  - 연결된 펜 : ** 개 
  - 열 항목: No, 펜 MAC 주소, 배터리 잔량, 펌웨어 버전, 업데이트 상태(최신/업데이트 필요).

- **노출 조건**: 사용자가 [일괄 업데이트 시작] 버튼을 클릭했을 때.

- **업데이트 로직**: '업데이트 필요' 상태인 펜이 1개 이상일 경우 [일괄 업데이트 시작] 버튼 활성화.
  - 일괄 업데이트 시작 버튼 클릭 시 : 개별 펜 업데이트 progress bar 로 진행 표시 
  - 펌웨어 업데이트 완료 시 : [일괄 업데이트 시작] 버튼 비활성화 
  - 펌웨어 업데이트 실패 시 : [일괄 업데이트 시작] 버튼 활성화 
  - 펌웨어 업데이트 진행 중 : [일괄 업데이트 시작] 버튼 비활성화 
  - [닫기] : 펌웨어 업데이트 모달 닫기 

- **화면 구성**:
  - 진행률 바: 각 펜 별 진행률 표시.
  - 펜별 상태: 각 펜의 업데이트 상태 표시(최신, 업데이트 필요, 업데이트 실패).

### 2.3 기기 초기화 확인 모달 (isResetModalOpen)

- **노출 조건**: [연결된 펜 전체 초기화] 버튼 클릭 시.

- **로직** : NeoStudio2Lite 데이터 동기화 후 펜 데이터 초기화 

- **안내 문구**: 
- "현재 N개의 펜이 연결되어 있습니다. \n 연결된 펜의 모든 데이터를 초기화하시겠습니까? \n 초기화 후 삭제된 데이터는 되돌릴 수 없습니다."

- **액션**: [취소] - 모달 닫기 / [초기화] - 초기화 프로세스 트리거(알림 후 종료).
- **종료 메시지** : 토스트 알림 "초기화가 완료되었습니다."

## 3. 예상 완료 시간 및 성능 로직 (Performance)

### 3.1 펌웨어 업데이트 예상 시간 산출

업데이트 시간은 연결된 펜의 개수와 네트워크 상태에 따라 동적으로 계산됩니다.

- **기본 공식**: `T = (N * S) / B + (N * Overhead)`
  - `N`: 연결된 펜 개수 (최대 30개)
  - `S`: 펌웨어 파일 크기 (평균 2MB)
  - `B`: 데이터 전송 속도 (블루투스/USB 대역폭 고려)
  - `Overhead`: 개별 펜 무팅 및 재부팅 시간 (약 5~10초)
- **표시 로직**: 펜 1개당 약 30초~1분 내외로 산출하여 "예상 소요 시간: 약 X분"으로 사용자에게 고지.

### 3.2 백그라운드 처리 (FAB 연동)

일괄 작업(업데이트/초기화) 진행 중 사용자가 모달을 닫아도 **하단 FAB(Floating Action Button)**을 통해 진행 상태를 지속적으로 피드백합니다.

## 4. 에러 처리 및 예외 상황 명세 (Error Handling)

| 상황            | 대응 시나리오                               | UI 피드백                                                           |
| :-------------- | :------------------------------------------ | :------------------------------------------------------------------ |
| **연결 해제**   | 업데이트 중 특정 펜의 연결이 끊긴 경우      | 리스트 내 해당 항목 '연결 실패' 빨간색 표시 및 재시도 버튼 노출     |
| **배터리 부족** | 펜 배터리가 20% 미만일 때                   | 업데이트 시작 전 "충전 필요" 경고 및 해당 펜 업데이트 버튼 비활성화 |
| **버전 충돌**   | 이미 최신 버전이거나 하위 버전 설치 시도 시 | "이미 최신 버전입니다" 알림 및 업데이트 생략                        |
| **초기화 실패** | 하드웨어 오류로 초기화 불가 시              | "일부 기기 초기화 실패" 리스트업 및 수동 초기화 가이드 제공         |

## 5. 접근성 및 UX 고려 사항 (Accessibility)

- **키보드 내비게이션**: 모달 오픈 시 Focus Trap을 적용하여 키보드 `Tab` 키로만 조작 가능하도록 함. `ESC` 키로 모달 닫기 지원.
- **색상 대비**: 위험 작업(초기화/삭제) 버튼은 `WCAG 2.1` 기준에 맞춘 고대비 빨간색(#EF4444) 사용.
- **스크린 리더**: 각 상태값(배터리 80%, 업데이트 필요 등)에 `aria-label`을 부여하여 시각 장애인이 상태를 정확히 인지하도록 설계.
- **피드백 가시성**: 성공/실패 시 텍스트 전용 알림뿐만 아니라 아이콘 체크/경고 표시를 병행하여 색맹 사용자 고려.
