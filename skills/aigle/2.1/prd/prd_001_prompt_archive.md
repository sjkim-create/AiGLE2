## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-02`
- **화면명**: Prompt 아카이브 (Prompt Archive)
- **URL 경로**: `/ai/prompt-archive`
- **대상 사용자**: AiGLE 개발자 및 관리자
- **접근 권한**: 'neolab.net' 계정인 id 교사 사용자에게만 노출
- **Figma Node ID**: `미정`
- **비즈니스 목적**: Prompt Studio에서 수행된 테스트 결과들을 영구 저장하고, 저장된 데이터를 필터링/정렬하여 관리하며, 선택된 데이터들을 기반으로 정밀 분석 리포트를 생성한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt 아카이브 메뉴에 진입하여 저장된 테스트 이력 목록을 확인한다.
2. 과목별 필터링 기능 또는 정렬 기능을 사용하여 특정 테스트 데이터를 찾는다.
3. 목록에서 결과 분류(정확도)나 오류 유형을 직접 수정하여 데이터를 정제한다.
4. 분석이 필요한 데이터들을 체크박스로 다중 선택한다.
5. 상단의 [데이터 분석 시작] 버튼을 클릭하여 분석 리포트 화면으로 이동한다.

## 2. 화면 요소 및 기능 (Elements & Functions)

_디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다._

### 2.1 상단 헤더 및 통계 영역

- **주요 요소:**
  - `타이틀/서브타이틀`: "Prompt 아카이브", "Prompt Studio 결과를 관리하고 데이터 분석 리포트를 생성합니다."
  - `데이터 분석 시작 버튼`: 다중 선택된 데이터의 개수를 표시하며 클릭 시 분석 프로세스 실행
- **기능 정책:**
  - 데이터가 1건 이상 선택되어야 버튼이 활성화됨
  - 선택되지 않은 상태에서 클릭 시 "분석할 데이터를 선택해 주세요." 알림 노출

### 2.2 툴바 및 필터 영역

- **주요 요소:**
  - `과목 필터 Select`: 전체, 국어, 수학, 영어, 과학, 사회 중 선택
  - `정렬 기준 Select`: 날짜순, 평가모드 순, 과제별 정렬 기준 선택
  - `정렬 방향 버튼`: 오름차순/내림차순 토글
  - `건수 표시`: 전체 데이터 수 및 현재 선택된 데이터 수 실시간 표시
- **기능 정책:**
  - 필터링 변경 시 하단 테이블 목록이 즉시 갱신됨

### 2.3 데이터 아카이브 테이블 (목록 영역)

- **주요 요소:**
  - `다중 선택 체크박스`: 헤더의 전체 선택 및 개별 로우 선택 가능
  - `결과 데이터 열`: 날짜, 과제명, 과목(배지), 평가모드, 사용 모델 표시
  - `결과 분류(정확도) Select`: ✅ 정확, ⚠️ 인접, ❌ 불일치 중 선택/수정
  - `오류 유형 Select`: OCR오류, Hallucination, Logic Error 등 상세 오류 유형 선택/수정
  - `비용 표시`: 미국 달러(USD) 및 원화(KRW) 환산 금액 표시
  - `삭제 버튼`: 개별 로우 삭제 기능
- **기능 정책:**
  - 수정된 결과 분류 및 오류 유형은 즉시 데이터에 반영됨
  - 삭제 시 "이 테스트 데이터를 삭제하시겠습니까?" 컨펌창 노출
  - 데이터가 없을 경우 "아카이브된 데이터가 없습니다. 'Prompt Studio'에서 결과를 저장해 주세요." 안내 문구 노출

## 3. 상태별 대응 (State & Interaction)

| 상태          | 발생 조건                        | UI/UX 동작 및 메시지                                                |
| :------------ | :------------------------------- | :------------------------------------------------------------------ |
| **Loading**   | 화면 진입 시 데이터 조회 중      | 테이블 영역 스켈레톤 UI 노출, 툴바 및 버튼 비활성화                 |
| **Success**   | 데이터 로드 완료 및 화면 진입 시 | 저장된 아카이브 리스트를 테이블 형식으로 출력                       |
| **Empty**     | 저장된 데이터가 하나도 없을 때   | 센터 정렬된 안내 아이콘과 "아카이브된 데이터가 없습니다." 문구 노출 |
| **Selected**  | 체크박스 선택 시                 | 해당 로우의 배경색 변경 및 상단 분석 버튼 내 숫자 갱신              |
| **Filtering** | 필터/정렬 조건 변경 시           | 애니메이션 없이 즉시 리스트 재배열                                  |

## 4. 비즈니스 로직 (Business Rules)

- **데이터 정렬**: 날짜 기준 내림차순(최신순)을 기본값으로 설정
- **환율 연산**: `[UPDATE v1.1]` 접속 시점의 환율을 실시간 반영하며, 동기화 요청 시 최신 환율을 재조회하여 표시
- **데이터 보존**: 사용자가 직접 삭제하기 전까지 LocalStorage 또는 DB를 통해 데이터 유지
- **분석 버튼 활성화**: 체크박스로 1건 이상 선택된 경우에만 [데이터 분석 시작] 버튼 활성화

---

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **데이터 대량 로드**: 아카이브 데이터가 많아질 경우 테이블 영역에 스크롤바가 생성되며, 헤더는 고정(Sticky) 처리되어 가독성 유지
- **삭제 후 빈 화면**: 마지막 남은 데이터를 삭제할 경우 즉시 'Empty' 상태 UI로 전환
- **브라우저 컨펌 취소**: 삭제 컨펌창에서 취소를 누를 경우 어떤 데이터 변경도 일어나지 않음
- **네트워크 에러**: 데이터 조회 실패 시 "데이터를 불러올 수 없습니다." 토스트 메시지 노출 및 재시도 버튼 제공
