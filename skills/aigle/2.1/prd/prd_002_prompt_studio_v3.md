## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-01`
- **화면명**: Prompt Studio
- **URL 경로**: `/ai/prompt-studio`
- **대상 사용자**: AiGLE 개발자
- **접근 권한**: 'neolab.net' 계정인 id 교사 사용자에게만 노출
- **Figma Node ID**: `미정`
- **비즈니스 목적**: 등록된 문제를 선택하고 학생 답안지를 업로드한 뒤, 시스템 프롬프트를 AI 모델에 전달하여 원하는 채점 결과를 테스트하고 관리할 수 있게 한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt Studio 화면에 진입하면 입력 파이프라인(왼쪽)과 AI 설정 사이드바(오른쪽)가 노출된다.
2. (Step 1) 등록 문항 선택: 등록된 과제 문항 중 하나를 선택한다. 해당 문항의 문제/모범답안/채점기준 데이터가 백엔드에서 로드된다.
3. (Step 2) 채점할 학생의 답안 이미지를 업로드한다.
4. (Step 3) 기본값으로 적용된 OCR 프롬프트와 채점 프롬프트(System Prompt)를 수정 또는 그대로 사용한다.
5. 오른쪽 사이드바에서 AI 모델(GPT/Gemini)과 파라미터를 설정한다.
6. 하단의 `AI 실행` 버튼을 눌러 채점을 진행하고, 결과를 확인한다.
7. 결과 카드의 `아카이브에 저장` 버튼을 눌러 결과를 보존한다.

## 2. 화면 요소 및 기능 (Elements & Functions)

*디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다.*

### 2.1 등록 문항 선택 영역 (Step 1)

- **주요 요소:**
  - `Select Box`: 채점할 테스트 문항 선택 / 초기값: 없음 / Placeholder: "문항을 선택해주세요"
  - `선택 문항 정보 패널`: 해당 문항의 [문제], [모범답안], [채점기준] 데이터 표시
- **기능 정책:**
  - 등록된 과제의 과제명이 Select Box에 노출되어야 함
  - 필수 입력 조건: 선택이 완료되어야 AI 실행 버튼 활성화 조건 1 충족
  - 문항을 변경하면 기존 결과 데이터 및 로드 상태가 초기화됨

### 2.2 학생 답안 이미지 업로드 영역 (Step 2)

- **주요 요소:**
  - `Drag & Drop Zone / File Input`: 클릭 시 파일 탐색기 열기 또는 드래그 업로드
  - `미리보기 이미지`: 업로드된 이미지 표시
  - `삭제 버튼`: 이미지 제거 및 초기 상태 복원
- **기능 정책:**
  - 필수 입력 조건: 이미지가 등록되어야 AI 실행 버튼 활성화 조건 2 충족
  - 허용 형식: 이미지(JPG, PNG) 및 PDF (최대 1개 파일)

### 2.3 시스템 프롬프트 설정 영역 (Step 3)

- **주요 요소:**
  - `OCR 프롬프트 TextArea`: 초기값: 기본 OCR 프롬프트
  - `채점 프롬프트 TextArea`: 초기값: 기본 채점 프롬프트
  - `[기본값으로 저장] 버튼`: 현재 내용을 DB/LocalStorage에 저장
- **기능 정책:**
  - 실시간 타이핑 편집 가능

### 2.4 AI 모델 및 파라미터 설정 (우측 사이드바)

- **주요 요소:**
  - `모델 선택 그룹`: GPT-4o mini, Gemini 3.1 Pro 등 버튼형 선택
  - `Parameter Sliders`: Temperature, Top-p, Max Tokens 등
- **기능 정책:**
  - 선택 모델에 따른 파라미터 항목 조건부 렌더링 (GPT vs Gemini)

### 2.5 AI 실행 버튼

- **주요 요소:**
  - `AI 실행 버튼`: 화면 하단 / 클릭 시 AI 채점 로직 실행
- **기능 정책:**
  - 활성화 조건: 문항 선택 AND 이미지 업로드 완료 시

### 2.6 채점 결과 카드 (ANALYSIS_RESULT)

- **주요 요소:**
  - `OCR 변환 텍스트`: 이미지에서 추출된 원문 텍스트 표시
  - `AI 채점 결과`: 등급 및 상세 피드백 노출
  - `비용/시간 정보`: 채점 소요 시간 및 USD/KRW 비용 정보
  - `[아카이브에 저장] 버튼`: 분석 결과 영구 저장
- **기능 정책:**
  - **이 영역은 AI 실행 성공 시 노출되는 메인 결과 화면임**
  - 저장 완료 시 버튼 비활성화 및 토스트 알림

## 3. 상태별 대응 (State & Interaction)

*각 상태는 `화면ID:상태ID` 형식으로 Figma 레이어와 매칭됩니다.*

### 3.1 AI-01:DEFAULT
- **상태 ID**: `DEFAULT`
- **발생 조건**: 화면 최초 진입 시
- **UI/UX**:
  - 기본 입력 폼 및 설정 사이드바 노출
  - 결과 카드(2.6 영역)는 숨김 상태

### 3.2 AI-01:LOADING
- **상태 ID**: `LOADING`
- **발생 조건**: AI 실행 버튼 클릭 후 서버 응답 대기 중
- **UI/UX**:
  - 버튼 텍스트 `AI 채점 중...` 변경 및 클릭 차단
  - 분석 영역에 로딩 애니메이션 또는 스켈레톤 UI 표시

### 3.3 AI-01:ANALYSIS_RESULT  <-- [2.6 영역 활성화]
- **상태 ID**: `ANALYSIS_RESULT`
- **발생 조건**: AI 실행 성공 후 분석 데이터(2.6 영역)가 화면에 표시될 때
- **UI/UX**:
  - **2.6 채점 결과 카드 영역**이 화면 하단/중앙에 노출됨
  - AI가 생성한 채점 데이터 및 OCR 텍스트가 바인딩됨
  - 성공 토스트 및 애니메이션 효과 적용

### 3.4 AI-01:ERROR
- **상태 ID**: `ERROR`
- **발생 조건**: 네트워크 단절 또는 API 오류 발생 시
- **UI/UX**:
  - 에러 팝업/토스트 노출
  - 실행 버튼 재활성화

## 4. 비즈니스 로직 (Business Rules)

- **유효성 검사**: 필수 입력(문항, 이미지) 누락 시 실행 버튼 Disabled
- **권한 정책**: 'neolab.net' 도메인 계정 사용자(교사)에게만 해당 화면 UI 및 분석 기능 허용
- **시스템 정책**: USD 비용을 KRW로 실시간 환율 적용하여 계산 (기본 1350원 기준)

---

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **데이터 누락**: AI 응답 필드 중 일부가 누락된 경우 "정보 없음"으로 표시
- **네트워크 에러**: API 호출 실패 시 기존 입력 데이터 유지하며 재시도 유도
