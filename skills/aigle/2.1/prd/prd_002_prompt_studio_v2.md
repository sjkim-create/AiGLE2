## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-01`
- **화면명**: Prompt Studio
- **URL 경로**: `/ai/prompt-studio`
- **대상 사용자**: AiGLE 개발자
- **접근 권한**: 'neolab.net' 계정인 id 교사 사용자에게만 노출
- **Figma Node ID**: `미정`
- **비즈니스 목적**: 등록된 문제를 선택하고 학생 답안지를 업로드한 뒤, 시스템 프롬프트를 AI 모델에 전달하여 원하는 채점 결과를 테스트하고 관리할 수 있게 한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt Studio 화면에 진입하면 입력 파이프라인(왼쪽)과 AI 설정 사이드바(오른쪽)가 노출된다.
2. (Step 1) 등록 문항 선택: 등록된 과제 문항 중 하나를 선택한다. 해당 문항의 문제/모범답안/채점기준 데이터가 백엔드에서 로드된다.
3. (Step 2) 채점할 학생의 답안 이미지를 업로드한다.
4. (Step 3) 기본값으로 적용된 OCR 프롬프트와 채점 프롬프트(System Prompt)를 수정 또는 그대로 사용한다.
5. 오른쪽 사이드바에서 AI 모델(GPT/Gemini)과 파라미터를 설정한다.
6. 하단의 `AI 실행` 버튼을 눌러 채점을 진행하고, 결과를 확인한다.
7. 결과 카드의 `아카이브에 저장` 버튼을 눌러 결과를 보존한다.

## 2. 화면 요소 및 기능 (Elements & Functions)

*디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다.*

### 2.1 등록 문항 선택 영역 (Step 1)

- **주요 요소:**
  - `Select Box`: 채점할 테스트 문항 선택 / 초기값: 없음 / Placeholder: "문항을 선택해주세요"
  - `선택 문항 정보 패널`: 해당 문항의 [문제], [모범답안], [채점기준] 데이터 표시
- **기능 정책:**
  - 등록된 과제의 과제명이 Select Box에 노출되어야 함
  - 필수 입력 조건: 선택이 완료되어야 AI 실행 버튼 활성화 조건 1 충족
  - 문항을 변경하면 기존 결과 데이터 및 로드 상태가 초기화됨
  - **인터랙션 정책**: 해당 영역 호버 또는 포커스 시 왼쪽 `sg-step-num` 강조 활성화

### 2.2 학생 답안 이미지 업로드 영역 (Step 2)

- **주요 요소:**
  - `Drag & Drop Zone / File Input`: 클릭 시 파일 탐색기 열기 또는 드래그 업로드
  - `미리보기 이미지`: 업로드된 이미지 표시
  - `삭제 버튼`: 이미지 제거 및 초기 상태 복원
- **기능 정책:**
  - 필수 입력 조건: 이미지가 등록되어야 AI 실행 버튼 활성화 조건 2 충족
  - 허용 형식: 이미지(JPG, PNG) 및 PDF (최대 1개 파일)
  - 새 이미지 업로드 시 기존 이미지 자동 대체

### 2.3 시스템 프롬프트 설정 영역 (Step 3)

- **주요 요소:**
  - `OCR 프롬프트 TextArea`: 초기값: 기본 OCR 프롬프트 / Placeholder: "프롬프트 내용이 없습니다"
  - `채점 프롬프트 TextArea`: 초기값: 기본 채점 프롬프트 / Placeholder: "프롬프트 내용이 없습니다"
  - `[기본값으로 저장] 버튼`: 현재 내용을 DB/LocalStorage에 저장
  - `[초기값으로 복구] 버튼`: 저장된 기본값 또는 시스템 초기값으로 복구
- **기능 정책:**
  - 실시간 타이핑 편집 가능
  - 저장/복구 시 각각 상태에 맞는 토스트 메시지 노출

### 2.4 AI 모델 및 파라미터 설정 (우측 사이드바)

- **주요 요소:**
  - `모델 선택 그룹`: GPT-4o mini, Gemini 3.1 Pro 등 버튼형 선택
  - `Paraemter Sliders`: Temperature, Top-p, Max Tokens 등
  - `Thinking Toggle`: GPT 모델 전용 활성화 스위치
- **기능 정책:**
  - 선택 모델에 따른 파라미터 항목 조건부 렌더링 (GPT vs Gemini)
  - API 스펙에 맞는 직관적인 조절 UI 제공

### 2.5 AI 실행 버튼

- **주요 요소:**
  - `AI 실행 버튼`: 화면 하단 / 클릭 시 AI 채점 로직 실행
- **기능 정책:**
  - 활성화 조건: 문항 선택 AND 이미지 업로드 완료 시
  - 클릭 시 Loading 상태로 전환되며 중복 클릭 방지 처리

### 2.6 채점 결과 카드 (ANALYSIS_COMPLETE)

- **주요 요소:**
  - `OCR 변환 텍스트`: 이미지에서 추출된 텍환 텍스트 표시
  - `AI 채점 결과`: 등급 및 피드백 포맷팅 노출
  - `비용/시간 정보`: 채점 소요 시간 및 USD/KRW 비용 (환율 연산 포함)
  - `[아카이브에 저장] 버튼`: 결과 영구 저장 및 저장 완료 시 비활성화
- **기능 정책:**
  - AI 분석 성공 시 애니메이션과 함께 노출
  - 저장 완료 시 버튼 텍스트 변경 및 재클릭 방지

## 3. 상태별 대응 (State & Interaction)

*각 상태는 `화면ID:상태ID` 형식으로 Figma 레이어와 매칭됩니다.*

### 3.1 AI-01:DEFAULT
- **상태 ID**: `DEFAULT`
- **발생 조건**: 화면 최초 진입 시
- **UI/UX**:
  - 입력 파이프라인 및 설정 사이드바 초기 상태 노출
  - AI 실행 버튼 비활성화 (Pending 상태)

### 3.2 AI-01:LOADING
- **상태 ID**: `LOADING`
- **발생 조건**: AI 실행 버튼 클릭 후 API 응답 대기 중
- **UI/UX**:
  - 버튼 텍스트가 `AI 채점 중...`으로 변경 및 Disabled
  - 분석 결과 영역에 로딩 인디케이터 또는 스켈레톤 노출

### 3.3 AI-01:ANALYSIS_COMPLETE
- **상태 ID**: `ANALYSIS_COMPLETE`
- **발생 조건**: AI 분석이 성공적으로 완료되어 결과 데이터가 수신되었을 때
- **UI/UX**:
  - **2.6 영역(채점 결과 카드)**이 화면에 활성화됨
  - 결과 데이터(텍스트, 등급, 비용 등)가 바인딩되어 표시
  - '📊 분석 완료' 애니메이션 및 토스트 피드백 제공

### 3.4 AI-01:ERROR
- **상태 ID**: `ERROR`
- **발생 조건**: API 호출 실패 또는 네트워크 오류 발생 시
- **UI/UX**:
  - 에러 메시지 토스트 노출 ("서버와 연결이 원활하지 않습니다...")
  - AI 실행 버튼 재활성화 (다시 시도 가능)

## 4. 비즈니스 로직 (Business Rules)

- **유효성 검사**: `selectedAssignId` 존재 여부 및 `studentImage` 파일 존재 여부 체크
- **권한 정책**: 'neolab.net' 도메인 계정을 가진 교사 권한 사용자에게만 페이지 접근 및 기능 허용
- **시스템 정책**: 
  - 비용 환율 계산: USD 값에 `exchangeRate`(기본 1350원, 실시간 변동 가능) 적용
  - 프롬프트 기본값은 LocalStorage를 우선 참조하며 없을 시 시스템 Default 값 로드

---

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **데이터 전송 오류**: 네트워크 차단 시 실행 버튼 초기화 및 재시도 안내
- **분석 형식 에러**: AI 응답이 규격화된 JSON이 아닐 경우 Raw Text 전체 노출
- **이미지 호환성**: PDF 업로드 시 첫 페이지만 이미지 처리하여 분석 시도
