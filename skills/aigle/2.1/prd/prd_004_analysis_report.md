## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-03`
- **화면명**: 분석 리포트 (Analysis Report)
- **URL 경로**: `/ai/analysis-report`
- **대상 사용자**: AiGLE 개발자 및 관리자
- **Figma Node ID**: `미정`
- **비즈니스 목적**: Prompt 아카이브에서 선택된 테스트 데이터들을 기반으로 정밀 통계 및 심층 분석 인사이트를 생성하고, 이를 시각화하여 제공하며 PDF 문서로 출력한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt 아카이브(AI-02) 화면에서 분석할 데이터들을 선택하고 [데이터 분석 시작] 버튼을 클릭한다.
2. 시스템이 전달받은 데이터를 기반으로 정확도, 비용, 오류 분포 등을 실시간 계산한다.
3. 분석 요약 카드, 인사이트 텍스트, 성능 비교 차트 및 상세 요약 테이블이 화면에 표시된다.
4. 사용자는 [PDF 리포트 출력] 버튼을 통해 분석 결과를 문서화한다.

## 2. 화면 요소 및 기능 (Elements & Functions)

*디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다.*

### 2.1 상단 헤더 및 컨트롤 영역

- **주요 요소:**
  - `타이틀`: "AI 채점 성능 분석 리포트"
  - `PDF 리포트 출력 버튼`: 클릭 시 브라우저 인쇄 기능을 호출하여 PDF 저장/출력 실행
- **기능 정책:**
  - 출력 시 헤더와 '돌아가기' 버튼 등 불필요한 UI는 제외하고 분석 리포트 내용만 포함됨 (Media Query: Print 대응)

### 2.2 요약 통계 대시보드 (Top Summary)

- **주요 요소:**
  - `분석 데이터 수 (Total Records)`: 선택된 전체 데이터 행 개수
  - `정확도 (Exact Match)`: 전체 중 '정확' 판정 데이터의 비율(%) 표시
  - `총 예상 비용 (Total Cost)`: 선택된 세션들의 총 사용 비용 합계 표시
  - `취약 영역 (Worst Area)`: 정답률이 가장 낮은 교과 영역 표시
  - `최다 빈출 오류 (Top Error)`: 가장 높은 빈도로 발생한 오류 유형 표시
- **기능 정책:**
  - **데이터 집계**: 아카이브 화면(AI-02)에서 선택되어 전달된 데이터 세트만을 대상으로 연산 수행
  - **정확도 계산**: `(결과 분류가 '정확'인 데이터 수 / 전체 데이터 수) × 100` (소수점 1자리까지 표시)
  - **비용 합산**: 각 데이터 행의 `원화(KRW)` 환산 금액을 누적 합계하여 천 단위 콤마(,) 포함 표시
  - **지표 추출**: 취약 영역 및 최다 빈출 오류는 각 데이터의 `과목` 및 `오류 유형` 필드를 빈도(Frequency) 기반으로 분석하여 자동 추출

### 2.3 심층 분석 인사이트 (Insights)

- **주요 요소 및 생성 로직:**
  - `1. 취약 영역 식별`:
    - **로직**: 카테고리별 정답률 중 최저 성능 영역을 추출
    - **문구**: "{최저 성능 영역} 영역에서 가장 낮은 정답률({해당 영역 정답률}%)을 보입니다. 이는 전체 평균 대비 {평균과의 격차}%p 낮은 수치로, 학습 데이터 보강이 필요합니다."
  - `2. 오류 원인 진단`:
    - **로직**: 전체 오류 데이터 중 가장 많이 발생한 오류 유형과 비중 계산
    - **문구**: "전체 오류의 약 {발생 비중}%가 '{최다 발생 오류}'에서 발생하고 있습니다. Vision 모델 해상도 및 전처리 점검이 권장됩니다."
  - `3. 평가 모드 격차`:
    - **로직**: '자율평가' 정답률과 '자동평가' 정답률 간의 차이 계산
    - **문구**: "자동평가 대비 자율평가 모드의 정답률이 {정답률 격차}%p 더 높습니다/낮습니다. 형식(Format) 기준에 따른 오답 발생 가능성을 시사합니다."
  - `4. 시간 대비 효율성`:
    - **로직**: 평균 처리 시간이 가장 긴 영역 추출
    - **문구**: "가장 시간이 오래 걸리는 '{최대 지연 영역}'(평균 {지연 시간}초) 영역의 정답률은 {해당 영역 정답률}%입니다."
- **기능 정책:**
  - 별도의 AI API 호출 없이, 요약 대시보드에서 산출된 통계 지표와 predefined 조건문을 조합한 룰 기반(Rule-based) 방식으로 텍스트 생성

### 2.4 영역별 정답률 차트

- **주요 요소:**
  - `가로 바 차트`: 카테고리별(교과) 정답률 시각화
- **기능 정책:**
  - **영역별 정답률 산출**: `(해당 교과에서 결과 분류가 '정확'인 데이터 수 / 해당 교과의 전체 데이터 수) × 100`으로 계산

### 2.5 평가 모드 비교 차트

- **주요 요소:**
  - `비교 차트`: 자동(Auto) 및 자율(Autonomous) 모드 정답률 시각적 대비
- **기능 정책:**
  - '자동평가'와 '자율평가' 그룹별로 각각의 정답률을 산출하여 비교 제공

### 2.6 모델 성능 상세 요약 테이블

- **주요 요소:**
  - `상세 요약 테이블`: 영역별(교과) 문항수, 정답률, 최다오류, 평균 처리 시간을 그리드 형태로 제공
- **기능 정책:**
  - 정답률 수치 옆에 해당 비율만큼 채워진 미니 프로그레스 바를 노출하여 시각적 직관성 제공
  - 각 교과별로 가장 많이 발생한 오류 유형(`OCR오류`, `논리 오류` 등)을 분석하여 노출
  - **평균 처리 시간**: `전체 처리 시간(Latency)의 합 / 문항 수` (소수점 2자리까지 표시)
  - **명칭 매핑**: 시스템 내부 영문 Key(Code)를 사용자 화면 표시용 한글 레이블로 일괄 치환하여 노출

## 4. 비즈니스 로직 (Business Rules)

- **데이터 정합성**: 모든 통계 지표는 Prompt 아카이브(AI-02)에서 전달된 스냅샷 데이터를 기준으로 하며, 리포트 화면 내에서의 개별 수정은 허용하지 않음
- **환율 연산 정책**: 아카이브 메뉴에서 반영된 원화 데이터를 그대로 사용하며, 리포트 화면 진입 시점에 추가적인 환율 API 호출이나 재연산은 수행하지 않음
- **데이터 그룹화**: '과목' 필드값을 기준으로 데이터를 분류하여 영역별 성능 지표(정답률, 처리 시간 등)를 산출
- **인사이트 엔진**: 별도의 AI API 호출 없이, 요약 대시보드에서 산출된 통계 지표와 predefined 조건문을 조합한 룰 기반(Rule-based) 방식으로 텍스트 생성
- **명칭 매핑**: 시스템 내부의 영문 Key(Code)를 사용자 화면 표시용 한글 레이블로 일괄 치환하여 노출

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **데이터 0건 진입**: 리포트 진입 전 아카이브 화면에서 차단하지만, 직접 URL 접근 등으로 진입 시에도 'Empty' 상태로 대응
- **오류 데이터 부재**: 모든 데이터가 '정확'일 경우 최다 빈출 오류 섹션에 "발생한 오류 없음" 또는 "-" 표시
- **지연 시간 누락**: `latency` 값이 없는 데이터는 평균 계산 시 제외하여 통계 왜곡 방지
- **단일 교과 데이터**: 교과가 1개만 존재할 경우 차트에 단일 바만 표시, 인사이트의 '취약 영역 식별'은 비교 대상 부재로 "비교 가능한 영역이 부족합니다." 표시
- **PDF 출력 실패**: 브라우저 팝업 차단 등으로 인쇄 기능이 호출되지 않을 경우 "팝업 차단을 해제해 주세요." 안내 노출
