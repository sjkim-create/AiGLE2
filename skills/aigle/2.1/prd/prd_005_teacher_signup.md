## 1. 화면 개요 (Overview)

- **화면 ID**: `JIN-02`
- **화면명**: 교사 회원가입
- **메뉴 경로**: `회원가입 > 회원정보 입력`
- **URL 경로**: `/join/teacher`
- **대상 사용자**: 교사 (신규 가입자)
- **Figma Node ID**: `미정`
- **비즈니스 목적**: 교사가 아이글 서비스에 가입하여 14일 무료 체험 계정을 생성할 수 있도록 한다. 이름, 이메일, 학교명 입력 및 약관 동의를 통해 회원 정보를 수집한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 이메일 인증(JIN-01)을 완료하고 회원정보 입력 화면에 진입한다.
2. 이름과 이메일은 이전 단계에서 인증된 값이 자동 입력되어 있다.
3. 사용자가 학교명 필드를 선택하여 학교 검색 모달(`pop-38`)에서 소속 학교를 검색하고 선택한다.
4. 필수 약관(이용약관, 개인정보 수집 및 이용)에 동의한다.
5. [다음] 버튼이 활성화되면 클릭하여 회원 가입 완료(JIN-03) 화면으로 이동한다.

## 2. 화면 구성

_사용자 액션이 필요한 기능만 기술합니다. 타이틀, 라벨 등 단순 표시는 Figma에서 확인합니다._

### 가입 유형 탭

**전체 공통:**
- [교사] / [학교] 탭을 클릭하여 **가입 유형 전환**
  - 성공: 선택한 유형에 맞는 입력 폼으로 전환
  - 조건: 항상 노출. 기본 선택은 [교사] 탭

### 회원정보 입력 폼

**전체 공통:**
- 이름 텍스트 필드에 **이름 입력**
  - 조건: 수정 안됨, 필수 입력 아님 (이전 단계 이메일 인증에서 확인된 값 자동 입력 가능)
- 이메일 텍스트 필드에 **이메일 주소 입력**
  - 조건: 수정 안됨, 필수 입력 아님 (이전 단계 이메일 인증에서 확인된 값 자동 입력 가능)
- 학교명 텍스트 필드에 학교명 입력 후 [검색] 버튼 클릭하여 **학교 검색**
  - 성공: 결과 목록이 표시되는 `pop-38` 모달 화면 노출, 사용자가 해당 학교 검색 후 선택
  - 실패: 검색 결과 없을 시 결과 없음 안내 메시지 노출
  - 조건: input 박스 선택 또는 [검색] 버튼 선택 시 `pop-38` 노출

### 약관 동의

**전체 공통:**
- [아래의 약관을 확인하고, 모두 동의 합니다.] 체크박스 클릭하여 **전체 약관 일괄 동의**
  - 성공: 하위 약관 항목 전체 체크 상태로 변경, [다음] 버튼 활성화
  - 조건: 하위 항목이 모두 체크된 경우 자동으로 전체 동의 체크 상태로 변경
- (필수) 이용약관에 동의합니다. 체크박스 클릭하여 **개별 동의**
  - 성공: 해당 항목 체크 상태로 변경
  - 조건: 필수 항목. 미동의 시 다음 단계 진행 불가, [다음] 버튼 비활성화
- (필수) 개인정보 수집 및 이용에 동의합니다. 체크박스 클릭하여 **개별 동의**
  - 성공: 해당 항목 체크 상태로 변경
  - 조건: 필수 항목. 미동의 시 다음 단계 진행 불가, [다음] 버튼 비활성화
- (선택) 이벤트, 서비스 안내수신에 동의합니다. 체크박스 클릭하여 **개별 동의**
  - 성공: 해당 항목 체크 상태로 변경
  - 조건: 선택 항목. 미동의 시에도 다음 단계 진행 가능
- 각 약관 항목 우측 [>] 버튼 클릭하여 **약관 상세 내용 확인**
  - 성공: 해당 약관 상세 내용 모달 페이지 노출

### 하단 버튼

**전체 공통:**
- [이전] 버튼 클릭하여 **이전 단계(이메일 인증)로 이동**
  - 성공: `JIN-01:교사회원가입이메일인증` 화면으로 이동
- [다음] 버튼 클릭하여 **다음 단계(회원 가입 완료)로 이동**
  - 성공: `JIN-03:회원가입완료` 화면으로 이동
  - 실패: 필수 항목(학교명, 필수 약관) 미입력/미동의 시 버튼 비활성화 또는 오류 안내
  - 조건: 필수 입력 항목(학교명) 입력 및 필수 약관 2개 동의 완료 시 활성화

## 3. 상태별 변화

_화면 상태에 따라 달라지는 부분입니다._

### JIN-02:교사회원가입회원정보입력 (초기상태)

- **상태 설명**: 화면 최초 진입 시의 기본 상태. [교사] 탭이 선택되어 있으며 입력 필드는 비어 있고, 약관은 모두 미체크 상태
- **전환 트리거**: 이전 단계(JIN-01:이메일 인증) 완료 → 현재 상태 진입

#### 하단 버튼

**전체 공통:**
- [다음] 버튼 **비활성화 상태**로 노출
  - 조건: 학교명 미입력 또는 필수 약관 미동의 상태

### JIN-02:교사회원가입회원정보입력 (입력완료)

- **상태 설명**: 필수 항목(학교명)이 입력되고 필수 약관 2개에 모두 동의한 상태
- **전환 트리거**: 학교명 입력 및 필수 약관 전체 동의 완료 → 현재 상태 진입

#### 하단 버튼

**전체 공통:**
- [다음] 버튼 **활성화 상태**로 변경
  - 성공: 버튼 클릭 시 회원 가입 완료(`JIN-03`) 화면으로 이동
  - 조건: 학교명 입력 완료 + 필수 약관 2개 동의 완료

## 4. 관련 화면

- **`JIN-01`**: 이메일 인증 - 회원가입 1단계. 이메일 인증 완료 후 JIN-02로 진입
- **`JIN-03`**: 회원 가입 완료 - 회원가입 3단계. JIN-02 완료 후 이동하는 최종 완료 화면
- **`pop-38`**: 학교 검색 모달 - 학교명 검색 및 선택을 위한 팝업 화면
