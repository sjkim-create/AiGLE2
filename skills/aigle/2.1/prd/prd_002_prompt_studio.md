## 1. 화면 개요 (Overview)

- **화면 ID**: `AI-01`
- **화면명**: Prompt Studio
- **URL 경로**: `/ai/prompt-studio`
- **대상 사용자**: AiGLE 개발자
- **접근 권한**: 'neolab.net' 계정인 id 교사 사용자에게만
- **Figma Node ID**: `미정`
- **비즈니스 목적**: 등록된 문제를 선택하고 학생 답안지를 업로드한 뒤, 시스템 프롬프트를 AI 모델에 전달하여 원하는 채점 결과를 테스트하고 관리할 수 있게 한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt Studio 화면에 진입하면 입력 파이프라인(왼쪽)과 AI 설정 사이드바(오른쪽)가 노출된다.
2. (Step 1) 등록 문항 선택: 등록된 과제 문항 중 하나를 선택한다. 해당 문항의 문제/모범답안/채점기준 데이터가 백엔드에서 로드된다.
3. (Step 2) 채점할 학생의 답안 이미지를 업로드한다.
4. (Step 3) 기본값으로 적용된 OCR 프롬프트와 채점 프롬프트(System Prompt)를 수정 또는 그대로 사용한다.
5. 오른쪽 사이드바에서 AI 모델(GPT/Gemini)과 파라미터를 설정한다.
6. 하단의 `AI 실행` 버튼을 눌러 채점을 진행하고, 결과를 확인한다.
7. 결과 카드의 `아카이브에 저장` 버튼을 눌러 결과를 보존한다.

## 2. 화면 요소 및 기능 (Elements & Functions)

_디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다._

### 2.1 등록 문항 선택 영역 (Step 1)

- **주요 요소:**
  - `Select Box`: 채점할 테스트 문항 선택 (예: '[국어] 문학 지문 분석', '[수학] 2차 방정식 풀이')
  - `선택 문항 정보 패널`: 선택 시 해당 문항의 [문제], [모범답안], [채점기준] 데이터가 백엔드에서 로드되어 표시됨
- **기능 정책:**
  - 등록된 과제의 과제명이 Select Box에 노출되어야 함
  - 필수 입력 조건: 선택이 완료되어야 AI 실행 버튼 활성화 조건 1 충족
  - 문항을 변경하면 기존 결과 데이터 및 로드 상태가 초기화됨
  - **인터랙션 정책**: 해당 영역에 마우스 호버 또는 Select Box에 포커스될 경우, 왼쪽의 `sg-step-num`이 활성화(색상 변경 및 강조 효과)되어 현재 작업 단계를 명확히 인지할 수 있도록 함

### 2.2 학생 답안 이미지 업로드 영역 (Step 2)

- **주요 요소:**
  - `Drag & Drop Zone / File Input`: 클릭 시 파일 탐색기 열림, 파일 드래그로 업로드 가능, 파일 첨부 시 이미지 미리보기로 전환
  - `미리보기 이미지`: 업로드된 학생 답안 이미지 표시 영역
  - `삭제 버튼`: 업로드된 이미지 제거 후 초기 상태로 복원
- **기능 정책:**
  - 필수 입력 조건: 이미지가 등록되어야 AI 실행 버튼 활성화 조건 2 충족
  - 허용 파일 형식: 이미지(JPG, PNG) 또는 PDF, 1개 파일만 업로드 가능
  - 이미지 변경 시 이전 이미지가 새 이미지로 대체됨
  - 삭제 버튼 클릭 시 업로드된 이미지가 삭제되고 초기 상태로 복원

### 2.3 시스템 프롬프트 설정 영역 (Step 3)

- **주요 요소:**
  - `OCR 프롬프트 TextArea`: 기본 프롬프트 노출. 내용이 없으면 "프롬프트 내용이 없습니다" 플레이스홀더 노출, 포커스 시 플레이스홀더 사라짐
  - `채점 프롬프트 (System Prompt) TextArea`: 기본 시스템 프롬프트 노출. 내용이 없으면 "프롬프트 내용이 없습니다" 플레이스홀더 노출, 포커스 시 플레이스홀더 사라짐
  - `[현재 내용을 기본값으로 저장] 버튼`: 현재 TextArea의 프롬프트 내용을 향후 기본값으로 DB(LocalStorage)에 저장
  - `[초기값으로 복구] 버튼`: 직전에 저장했던 기본값으로 텍스트 영역 내용 복구 (저장 이력이 없으면 시스템 최초 초기값 로드)
- **기능 정책:**
  - 사용자가 직접 타이핑하여 내용을 실시간으로 편집 가능
  - **기본값 저장 시**: "현재 프롬프트 내용이 기본값으로 저장되었습니다. 이후 서비스 이용 시 해당 내용이 기본값으로 자동 반영됩니다." 토스트 메시지 노출
  - **초기값 복구 시**: "프롬프트 내용이 초기화 되었습니다." 토스트 메시지 노출

### 2.4 AI 모델 선택 및 파라미터 설정 (우측 사이드바)

- **주요 요소 (모델 선택):**
  - `GPT 모델` 그룹: GPT-4o mini, GPT-5.3 Instant, GPT-5.4 Thinking, GPT-5.4 Pro 버튼
  - `Gemini 모델` 그룹: Gemini 3.1 Pro, Gemini 3.1 Flash-Lite 버튼

- **주요 요소 (파라미터 설정):**
  - `temperature` (Range 0~1): 창의성/일관성 조절
  - `top_p` (Range 0~1): 단어 선택 폭 조절
  - `max_tokens` (Range 100~8192): GPT / Gemini에 따라 레이블 이름 분기
  - `thinking` (토글): GPT 모델 한정 노출 (boolean 활성/비활성 처리)
  - `thinking_config` (버튼): Gemini 모델 한정 노출 (Minimal, Low, Medium, High 4단계 버튼형, 기본값: Medium)
- **기능 정책:**
  - 선택한 모델(GPT vs Gemini)에 따라 파라미터 항목이 조건부 렌더링됨
  - 선택한 모델의 API 스펙에 대응하는 파라미터를 직관적으로 조절할 수 있게 구성

### 2.5 AI 실행 버튼

- **주요 요소:**
  - `AI 실행 버튼`: 화면 하단에 위치, 모든 필수 조건 충족 시 활성화
- **기능 정책:**
  - 활성화 조건: `selectedAssignId`(문항 선택)와 `studentImage`(답안 이미지)가 모두 존재할 때
  - 클릭 시 선택된 AI 모델과 파라미터 설정을 기반으로 채점 실행

### 2.6 채점 결과 카드 (분석 완료 상태)

- **주요 요소:**
  - `OCR 변환 텍스트`: 이미지에서 문자로 변환된 Raw Text
  - `AI 채점 결과`: 평가 등급, 피드백을 포맷팅된 형태(원문)로 표시
  - `채점 시간`: 예) 2.42s
  - `사용 토큰 정보`: 입력 / 출력 / 총합 수치 출력
  - `예상 비용`: USD 기반 소수점 표시, 버튼 클릭 시 원화(KRW)로 실시간 환율 연산하여 노출 (↻ 버튼으로 갱신)
  - `아카이브에 저장 버튼`: 결과 창 우측 상단에 위치
- **기능 정책:**
  - AI 채점 완료 시 애니메이션과 함께 왼쪽 파이프라인 하단에 '📊 분석 완료' 결과 카드 노출
  - `아카이브에 저장` 클릭 시 분석 내용 저장, 버튼 텍스트가 `✓ 아카이브 저장됨`으로 변경되며 Disabled 처리
  - 저장 완료 시 "테스트 결과가 아카이브에 저장되었습니다." 토스트 메시지 출력

## 3. 상태별 대응 (State & Interaction)

| 상태                    | 발생 조건                | UI/UX 동작 및 메시지                                                                             |
| :---------------------- | :----------------------- | :----------------------------------------------------------------------------------------------- |
| **Pending (대기)**      | 화면 최초 진입           | 입력되지 않은 정보(문항 또는 답안)가 있을 경우 AI 실행 버튼 비활성화                             |
| **Loading (채점 중)**   | AI 실행 버튼 클릭 시     | 버튼 텍스트가 `AI 채점 중...`으로 변경, 중복 클릭 방지를 위해 Disabled 처리                      |
| **Success (결과 반환)** | AI API 호출 응답 완료 시 | 애니메이션과 함께 왼쪽 파이프라인 하단에 '📊 분석 완료' 결과 카드 노출                           |
| **Error (실패)**        | AI API 호출 실패 시      | "서버와 연결이 원활하지 않습니다. 다시 시도해 주세요." 토스트 메시지 노출, AI 실행 버튼 재활성화 |

## 4. 비즈니스 로직 (Business Rules)

- **입력 유효성 판단**: 좌측의 `selectedAssignId`(문항)와 `studentImage`(답안 이미지)가 모두 존재할 때만 AI 실행 버튼 활성화
- **모의 데이터 채점 연산**: 현재 버전에서는 MockData 기반의 Promise(setTimeout 2초 딜레이) 처리하여 결과값 노출 (추후 실제 AI API 연동 예정)
- **비용 환율 환산**: `$ 0.00085`와 같이 반환된 USD 기준 비용에 `exchangeRate`(기본 1350원 기준, 새로고침 시 난수 반영)를 곱하여 원화(KRW)로 노출
- **프롬프트 기본값 관리**: 사용자가 저장한 프롬프트 기본값은 LocalStorage에 보관하여 다음 접속 시 자동 반영
- **문항 변경 시 초기화**: 문항을 변경하면 기존 결과 데이터 및 로드 상태가 모두 초기화됨

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **데이터 전송 오류**: AI 분석 중 서버 통신이 끊길 경우 "서버와 연결이 원활하지 않습니다. 다시 시도해 주세요." 토스트 노출
- **분석 결과 형식 불일치**: AI가 규격화된 형식을 반환하지 않을 경우, 'AI 채점 결과 (Raw)' 영역에 수신된 텍스트 전체 노출
- **이미지 업로드 실패**: 지원하지 않는 파일 형식이거나 용량 초과 시 파일 선택을 취소하고 경고 문구 출력
- **프롬프트 저장 이력 없음**: `[초기값으로 복구]` 클릭 시 저장 이력이 없으면 시스템 최초 초기값을 로드
- **등록 문항 없음**: 등록된 과제가 없을 경우 Select Box에 "등록된 문항이 없습니다" 안내 노출
