## 1. 화면 개요 (Overview)

- **화면 ID**: `SET-01`
- **화면명**: 환경설정 (Settings)
- **대상 사용자**: 로그인된 사용자 (AiGLE 서비스 이용자)
- **Figma Node ID**: `{node-id}`
- **비즈니스 목적**: 사용자가 채점 전 환경설정을 셋팅할 수 있도록 하여 채점이 지연되는 것을 방지한다. 연결된 펜 디바이스의 펌웨어 및 소프트웨어를 직접 관리하고, 기기 초기화 등 주요 디바이스 운영 작업을 수행할 수 있도록 지원한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 왼쪽 사이드바 상단 프로필 아이콘을 클릭하여 드롭다운 메뉴를 열고 [환경설정]을 선택한다.
2. 시스템이 설정 전용 사이드바(내 정보 / 고객센터 / 환경설정)로 전환하고 환경설정 메인 화면을 카드형 레이아웃으로 표시한다.
3. 사용자가 원하는 기능 카드(AiGLE Connect 관리 / 펜 펌웨어 업데이트 / 기기 초기화)에서 액션을 수행한다.
4. 각 액션에 따라 다운로드·업데이트·초기화가 처리되고, 결과가 화면 또는 토스트 알림으로 피드백된다.

## 2. 화면 요소 및 기능 (Elements & Functions)

*디자인 속성(색상, 간격)은 Figma에서 확인합니다. 여기서는 데이터와 기능 위주로 기술합니다.*

### 2.1 AiGLE Connect 관리 카드

- **주요 요소:**
  - `현재 버전`: 텍스트 표시 / 설치 여부에 따라 "설치되지 않음" 또는 버전 문자열(예: `1.2.3`) 출력
  - `최신 버전`: 텍스트 표시 / 서버에서 조회한 최신 버전 문자열 출력
  - `최신 버전 다운로드` 버튼: 클릭 시 AiGLE Connect 최신 버전 설치 파일 다운로드

- **기능 정책:**
  - **버튼 활성화 조건**: 소프트웨어가 설치되지 않았거나(`설치되지 않음`), 현재 버전이 최신 버전보다 낮은 경우
  - **버튼 비활성화 조건**: 현재 버전과 최신 버전이 동일한 경우(이미 최신 상태)
  - 상태별 카드 메시지 표시

| 상태 | 현재 버전 표시 | 최신 버전 표시 | 버튼 상태 |
| :--- | :--- | :--- | :--- |
| 설치되지 않음 | "설치되지 않음" | X.X.X | 활성화 |
| 업데이트 필요 | X.X.X | X.X.X (상위 버전) | 활성화 |
| 최신 버전 설치됨 | X.X.X | X.X.X (동일) | 비활성화 |

### 2.2 펜 펌웨어 업데이트 카드 및 모달 

**[카드 영역]**

- **주요 요소:**
  - `연결된 펜`: 현재 연결된 펜 총 개수 표시 (예: `30개`)
  - `업데이트 필요` 상태 배지: 업데이트가 필요한 펜이 1개 이상인 경우 상태 표시
  - `펜 목록 및 펌웨어 업데이트` 버튼: 클릭 시 펌웨어 업데이트 모달 오픈

**[모달 영역]**

- **주요 요소:**
  - `연결된 펜 총 개수` 헤더 정보
  - 펜 목록 테이블

| 열 항목 | 설명 |
| :--- | :--- |
| No | 순번 |
| 펜 MAC 주소 | 고유 식별자 |
| 배터리 잔량 | % 단위 표시 |
| 펌웨어 버전 | 현재 설치된 버전 |
| 업데이트 상태 | `최신` / `업데이트 필요` / `업데이트 실패` |

  - `일괄 업데이트 시작` 버튼
  - `닫기` 버튼

- **기능 정책:**
  - `일괄 업데이트 시작` 버튼은 '업데이트 필요' 상태인 펜이 1개 이상일 경우에만 활성화
  - 업데이트 진행 중에는 버튼 비활성화
  - 전체 업데이트 완료 시 버튼 비활성화
  - 업데이트 실패 시 버튼 재활성화 (재시도 가능)
  - 업데이트 진행 시 각 펜 별 Progress Bar 표시
  - [닫기] 클릭 시 모달 닫기. 단, 업데이트 진행 중 모달을 닫아도 하단 FAB를 통해 진행 상태 지속 피드백
  - **예상 소요 시간 산출 공식**: `T = (N × S) / B + (N × Overhead)` (펜 1개당 약 30초~1분 내외로 산출, "예상 소요 시간: 약 X분"으로 사용자에게 고지)

### 2.3 기기 초기화 카드 및 확인 모달 

**[카드 영역]**

- **주요 요소:**
  - 위험성 안내 문구
  - `연결된 펜 전체 초기화` 버튼 (위험 작업 버튼, WCAG 2.1 기준 고대비 빨간색 `#EF4444` 사용)

**[모달 영역]**

- **주요 요소:**
  - 안내 문구: `"현재 N개의 펜이 연결되어 있습니다. 연결된 펜의 모든 데이터를 초기화하시겠습니까? 초기화 후 삭제된 데이터는 되돌릴 수 없습니다."`
  - `취소` 버튼: 모달 닫기
  - `초기화` 버튼: 초기화 프로세스 실행

- **기능 정책:**
  - 초기화 전 NeoStudio2Lite와 데이터 동기화 선행 처리
  - 초기화 완료 시 토스트 알림: `"초기화가 완료되었습니다."`

## 3. 상태별 대응 (State & Interaction)

| 상태 | 발생 조건 | UI/UX 동작 및 메시지 |
| :--- | :--- | :--- |
| **Loading** | 펌웨어 업데이트 진행 중 | 각 펜별 Progress Bar 표시, [일괄 업데이트 시작] 버튼 비활성화 |
| **Success - 펌웨어** | 전체 펜 업데이트 완료 | 업데이트 상태 `최신`으로 변경, [일괄 업데이트 시작] 버튼 비활성화 |
| **Success - 초기화** | 초기화 프로세스 정상 완료 | 모달 닫힘, 토스트 알림 "초기화가 완료되었습니다." 표시 |
| **Empty (펜 없음)** | 연결된 펜이 0개인 경우 | 펜 목록 "연결된 펜이 없습니다." 안내, 업데이트/초기화 버튼 비활성화 |
| **Error - 연결 해제** | 업데이트 중 특정 펜 연결 끊김 | 해당 펜 항목 빨간색 `연결 실패` 표시 및 재시도 버튼 노출 |
| **Error - 배터리 부족** | 펜 배터리 20% 미만 | 업데이트 시작 전 "충전 필요" 경고, 해당 펜 업데이트 버튼 비활성화 |
| **Error - 버전 충돌** | 이미 최신 버전 또는 하위 버전 설치 시도 | "이미 최신 버전입니다." 알림 및 업데이트 생략 처리 |
| **Error - 초기화 실패** | 하드웨어 오류로 초기화 불가 | "일부 기기 초기화 실패" 목록 표시 및 수동 초기화 가이드 제공 |

## 4. 비즈니스 로직 (Business Rules)

- **업데이트 가능 여부 판단**: 펜의 현재 펌웨어 버전과 서버의 최신 버전을 비교하여 `업데이트 필요` 상태 부여
- **일괄 업데이트 대상**: `업데이트 필요` 상태인 펜에 한해 순차적으로 업데이트 실행
 (확인 필요)- **업데이트 예상 시간 산출**: `T = (N × S) / B + (N × Overhead)`, N = 펜 개수(최대 30개), S = 펌웨어 파일 크기(평균 2MB), Overhead = 개별 펜 마운팅 및 재부팅 시간(약 5~10초)
- **초기화 사전 처리**: AiGLE Connect와 데이터 동기화 완료 후 펜 초기화 진행
 (확인 필요)- **배터리 정책**: 펜 배터리가 20% 미만이면 해당 펜은 업데이트 대상에서 제외하고 경고 표시
- **백그라운드 처리**: 일괄 업데이트·초기화 진행 중 모달 닫아도 하단 FAB(Floating Action Button)으로 진행 상태 지속 피드백

## 5. 예외 및 엣지 케이스 (Edge Cases)

- **연결된 펜 0개**: 펌웨어 업데이트 모달 내 빈 목록 안내 문구 노출, 업데이트·초기화 버튼 비활성화
- **업데이트 중 펜 연결 해제**: 해당 펜 항목을 `연결 실패` 상태로 표시, 나머지 펜은 업데이트 계속 진행
 (확인 필요)- **배터리 20% 미만 펜**: 업데이트 시작 버튼 클릭 전 사전 경고, 해당 펜만 업데이트 제외
- **펌웨어 최신 버전 API 응답 실패**: 버전 정보 로드 실패 안내 및 재시도 유도
- **초기화 부분 실패**: 실패한 기기 목록 리스트업, 수동 초기화 가이드 URL 또는 문서 제공
- **AiGLE Connect 버전 API 미응답**: 버전 정보 "확인 불가" 표시, 다운로드 버튼은 활성화 유지

## 6. 접근성 (Accessibility)

- **키보드 내비게이션**: 모달 오픈 시 Focus Trap 적용, `Tab` 키로만 내부 요소 순환, `ESC` 키로 모달 닫기 지원
- **색상 대비**: 위험 작업(초기화/삭제) 버튼은 WCAG 2.1 기준 고대비 빨간색(`#EF4444`) 사용
- **스크린 리더**: 배터리 잔량, 업데이트 상태 등 각 상태값에 `aria-label` 부여
- **색맹 대응**: 성공/실패 피드백 시 텍스트 + 아이콘(체크/경고) 병행 표시
